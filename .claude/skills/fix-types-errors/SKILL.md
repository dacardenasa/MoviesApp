name: fix-types-errors
description: Helps users fix type errors in their TypeScript code. Use when the user encounters type-related issues or wants to understand how to resolve them.
---

When fixing type errors, always include:
1. **Identify the error**: Clearly state what the type error is and where it occurs in the code. Use analogies to real-life situations to make it relatable. For example, you could compare a type error to trying to fit a square peg into a round hole – it simply doesn't fit because it's not the right shape.
2. **Explain the cause**: Describe why the type error is happening. This could be due to a mismatch between expected and actual types, missing properties, or incorrect type annotations. You could compare this to trying to use a key that doesn't match the lock – it won't work because it's not designed for that purpose.
3. **Provide a solution**: Offer a clear and concise solution to fix the type error. This could involve changing the type annotations, adding missing properties, or using type assertions. You could compare this to finding the right key for the lock – once you have the correct key, it will fit perfectly and allow you to open the door.
4. **Highlight a gotcha**: Point out any common mistakes or misconceptions that could lead to similar type errors in the future. This is like warning someone about a common pitfall when using a tool – for example, if you're using a hammer, you might warn them to be careful not to hit their fingers, which is a common mistake that can happen if they're not paying attention.

Keep explanations conversational. For complex concepts, use multiple analogies.
